# <a name="replenishment"></a><span data-ttu-id="66db8-103">التزويد</span><span class="sxs-lookup"><span data-stu-id="66db8-103">Replenishment</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="66db8-104">يصف هذا الموضوع استراتيجيات التزويد المتوفرة للمستودعات التي تستخدم الوظيفة المتوفرة في إدارة المستودعات.</span><span class="sxs-lookup"><span data-stu-id="66db8-104">This topic describes the replenishment strategies that are available for warehouses which use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="66db8-105">لا تنطبق المعلومات على حل التخزين المتوفر في إدارة المخزون.</span><span class="sxs-lookup"><span data-stu-id="66db8-105">The information doesn't apply to the warehousing solution that is available in Inventory management.</span></span> <span data-ttu-id="66db8-106">تتوفر استراتيجيات التزويد الثلاث التالية:</span><span class="sxs-lookup"><span data-stu-id="66db8-106">The following replenishment strategies are available:</span></span>

-   <span data-ttu-id="66db8-107">**التزويد على أساس الطلب** – تنشئ هذه الاستراتيجية عمل تزويد للأوامر أو الأحمال الصادرة أو إذا لم يتوفر المخزون عندما تقوم الموجة بإنشاء العمل.</span><span class="sxs-lookup"><span data-stu-id="66db8-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="66db8-108">على سبيل المثال، يمكن إنشاء عمل التزويد إذا كانت الكمية المطلوبة لأمر مبيعات غير متوفرة عند معالجة الموجة.</span><span class="sxs-lookup"><span data-stu-id="66db8-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
-   <span data-ttu-id="66db8-109">**الحد الأدنى/الحد الأقصى لعملية التزويد** – تستخدم هذه الاستراتيجية الحد الأدنى والحد الأقصى لحدود التخزين لتحديد الوقت الذي يجب فيه تزويد المواقع.</span><span class="sxs-lookup"><span data-stu-id="66db8-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="66db8-110">معايير الصنف والموقع تعرف المخزون الذي يتم تقييمه للتزويد.</span><span class="sxs-lookup"><span data-stu-id="66db8-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="66db8-111">تُعد قوالب الحد الأدنى/الحد الأقصى لعملية التزويد الآلية الرئيسية للحفاظ على المستويات المثلى في مواقع الانتقاء.</span><span class="sxs-lookup"><span data-stu-id="66db8-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="66db8-112">لضمان توفر كمية انتقاء كافية في واجهة المخزون لتلبية طلب الموجة، يمكنك استخدام التزويد على أساس الطلب كمكمل بين دورات الحد الأدنى/الحد الأقصى لعملية التزويد.</span><span class="sxs-lookup"><span data-stu-id="66db8-112">To ensure that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
-   <span data-ttu-id="66db8-113">**تزويد طلب الحمل‬** – تجمع هذه الاستراتيجية طلب عدة أحمال وتنشئ عمل التزويد المطلوب لتخزين مواقع الانتقاء ذات الصلة.</span><span class="sxs-lookup"><span data-stu-id="66db8-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="66db8-114">تساعد هذه الاستراتيجية على ضمان إمكانية انتقاء الأحمال التي تم إنشاؤها في المستودع بعد إصدارها.</span><span class="sxs-lookup"><span data-stu-id="66db8-114">This strategy helps to ensure that the loads that are created can be picked in the warehouse after they are released.</span></span>

<span data-ttu-id="66db8-115">تنشئ الاستراتيجيات الثلاث كلها عمل التزويد، استنادًا إلى قالب التزويد.</span><span class="sxs-lookup"><span data-stu-id="66db8-115">All three strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="66db8-116">تزويد طلب الموجة</span><span class="sxs-lookup"><span data-stu-id="66db8-116">Wave demand replenishment</span></span>

<span data-ttu-id="66db8-117">ينشئ تزويد طلب الموجة عمل تزويد، استنادًا إلى الطلب، إذا كانت الكمية المطلوبة للأوامر الإنتاج أو كانبان أو الأحمال أو الأوامر الصادرة غير متوفرة عندما تقوم الموجة بإنشاء العمل.</span><span class="sxs-lookup"><span data-stu-id="66db8-117">Wave demand replenishment creates replenishment work, based on demand, if the quantity that is required for production orders, kanbans, outbound orders or loads isn't available when a wave creates work.</span></span> <span data-ttu-id="66db8-118">يحتوي قالب التزويد على معلومات حول معايير الصنف ووحدة القياس وزيادة الطلب والموقع.</span><span class="sxs-lookup"><span data-stu-id="66db8-118">The replenishment template contains information about the item criteria, the unit of measure, the demand increment, and the location.</span></span> 

<span data-ttu-id="66db8-119">يتم استخدام توجيهات الموقع لتحديد الموقع الذي يجب تزويده.</span><span class="sxs-lookup"><span data-stu-id="66db8-119">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="66db8-120">ويمكنك ربط توجيهات الموقع هذه بقالب التزويد باستخدام حقل **كود التوجيه‬**.</span><span class="sxs-lookup"><span data-stu-id="66db8-120">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="66db8-121">إذا لم يتم تعيين حقل **كود التوجيه**، فسيتم استخدام الاستعلامات لتحديد توجيه الموقع الذي يجب استخدامه.</span><span class="sxs-lookup"><span data-stu-id="66db8-121">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="66db8-122">لاحظ أنه إذا لم يتم تحديد كود توجيه في قالب التزويد، وإذا كان هناك كود توجيه لتوجيه الموقع، فسيتم تجاهل توجيه الموقع، حتى لو كان الاستعلام على توجيه الموقع صحيحًا.</span><span class="sxs-lookup"><span data-stu-id="66db8-122">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="66db8-123">يتم استخدام توجيهات موقع الانتقاء لتحديد مكان الحصول على المخزون لإجراء التزويد.</span><span class="sxs-lookup"><span data-stu-id="66db8-123">Pick location directives are used to determine where to get inventory for the replenishment.</span></span> 

<span data-ttu-id="66db8-124">بالإضافة إلى إنشاء قالب، يجب تحديد بعض إعدادات التزويد في قالب الموجة.</span><span class="sxs-lookup"><span data-stu-id="66db8-124">In addition to creating a template, you must specify some replenishment settings in the wave template.</span></span> <span data-ttu-id="66db8-125">يجب أن يتضمن قالب الموجة خطوة موجة لعملية التزويد التي يتم تشغيلها فقط إذا لم تنجح عملية توزيع أحد الأصناف.</span><span class="sxs-lookup"><span data-stu-id="66db8-125">The wave template should contain a wave step for replenishment that is run only if allocation of an item isn't successful.</span></span> <span data-ttu-id="66db8-126">تستخدم خطوة إعداد موجة التزويد هذه كود خطوة الموجة لتحديد قالب التزويد الذي يجب استخدامه.</span><span class="sxs-lookup"><span data-stu-id="66db8-126">This replenishment wave step uses a wave step code to determine which replenishment template should be used.</span></span> <span data-ttu-id="66db8-127">بالإضافة إلى وجود خطوة موجة لإجراء عملية التزويد، يجب أن تتأكد من تحديد **التزويد** في مقطع **الأساليب** في قالب الموجة.</span><span class="sxs-lookup"><span data-stu-id="66db8-127">In addition to having a wave step for replenishment, you need to make sure that **Replenish** is selected in the **Methods** section of the wave template.</span></span> 

<span data-ttu-id="66db8-128">تتضمن صفحة **قالب التزويد** خانة الاختيار **السماح لطلب الموجة باستخدام الكميات غير المحجوزة‬**.</span><span class="sxs-lookup"><span data-stu-id="66db8-128">The **Replenishment template** page includes an **Allow wave demand to use unreserved quantities** check box.</span></span> <span data-ttu-id="66db8-129">يجب تحديد خانة الاختيار هذه إذا كنت تريد السماح لعملية التزويد على أساس الطلب بخصم الكميات غير المحجوزة من العمل الذي تم إنشاؤه من قالب التزويد المحدد.</span><span class="sxs-lookup"><span data-stu-id="66db8-129">You should select this check box if you want to allow demand replenishment to deduct unreserved quantities from work that is generated from the selected replenishment template.</span></span> <span data-ttu-id="66db8-130">لتمكين قوالب التزويد على أساس الطلب من استخدام هذا المنطق، حدد خانة الاختيار لكل قالب تزويد موجود.</span><span class="sxs-lookup"><span data-stu-id="66db8-130">To enable demand replenishment templates to use this logic, select this check box for every existing replenishment template.</span></span> <span data-ttu-id="66db8-131">عند تشغيل التزويد على أساس الطلب في المستودع، سيتم خصم الطلب من عمل التزويد الحالي الذي يتضمن كميات غير محجوزة، إذا كان العمل ناشئًا من قوالب التزويد حيث تم تمكين خانة الاختيار **السماح لطلب الموجة باستخدام الكميات غير المحجوزة‬**.</span><span class="sxs-lookup"><span data-stu-id="66db8-131">When demand replenishment is triggered in the warehouse, it will deduct the demand from existing replenishment work that has unreserved quantities, if the work originates from replenishment templates where the **Allow wave demand to use unreserved quantities** check box is selected.</span></span>


<span data-ttu-id="66db8-132">يتم دعم التزويد على أساس الطلب لأوامر المبيعات وأوامر التحويل وأوامر الإنتاج وكانبان.</span><span class="sxs-lookup"><span data-stu-id="66db8-132">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="66db8-133">الحد الأدنى/الحد الأقصى لعملية التزويد</span><span class="sxs-lookup"><span data-stu-id="66db8-133">Min/Max replenishment</span></span>
<span data-ttu-id="66db8-134">في الحد الأدنى/الحد الأقصى لعملية التزويد، يتم تزويد المخزون بحيث يتواجد بين الحدين الأدنى والأقصى اللذين تم تعيينهما.</span><span class="sxs-lookup"><span data-stu-id="66db8-134">In Min/Max replenishment, stock is replenished so that it's between the minimum and maximum limits that have been set.</span></span> <span data-ttu-id="66db8-135">تحدث هذه العملية عادةً مرة واحدة كل يوم لضمان تعبئة كافة مواقع الانتقاء إلى الحد الأقصى للمستوى الذي كانت عليه قبل بدء الانتقاء.</span><span class="sxs-lookup"><span data-stu-id="66db8-135">Typically, this process occurs one time every day to ensure that all picking locations are filled to the maximum level before picking starts.</span></span> 

<span data-ttu-id="66db8-136">يتم تعيين الحد الأدنى والحد الأقصى للمبالغ في قالب التزويد.</span><span class="sxs-lookup"><span data-stu-id="66db8-136">The minimum and maximum amounts are set in a replenishment template.</span></span> <span data-ttu-id="66db8-137">وهناك العديد من الإعدادات الأخرى في القالب تشبه الإعدادات في القوالب التي يتم استخدامها في تزويد طلب الموجة.</span><span class="sxs-lookup"><span data-stu-id="66db8-137">Many of the other settings in the template resemble the settings in templates that are used in Wave demand replenishment.</span></span> <span data-ttu-id="66db8-138">يجب أن يحتوي القالب على بند واحد لكل صنف وموقع.</span><span class="sxs-lookup"><span data-stu-id="66db8-138">The template should contain one line for each item and location.</span></span> <span data-ttu-id="66db8-139">عند تشغيل التزويد باستخدام وظيفة دفعية، سيقوم Finance and Operations بتقييم ما إذا كان التزويد مطلوبًا، في التسلسل الذي تم تنظيم البنود به.</span><span class="sxs-lookup"><span data-stu-id="66db8-139">When you run replenishment by using the batch job, Finance and Operations evaluates whether replenishment is required, in the sequence that the lines are organized in.</span></span> 

<span data-ttu-id="66db8-140">لاحظ أن استراتيجية الحد الأدنى/الحد الأقصى لعملية التزويد لا يمكنها تزويد موقع فارغ إلا إذا تم تعيين الموقع كموقع ثابت للصنف.</span><span class="sxs-lookup"><span data-stu-id="66db8-140">Note that the Min/Max replenishment strategy can't replenish an empty location unless the location is set as the fixed location for the item.</span></span> <span data-ttu-id="66db8-141">إذا لم يكن الموقع الذي يجب تزويده عبارة عن موقع ثابت، فسيتعذر تحديد الصنف الذي يجب تزويده.</span><span class="sxs-lookup"><span data-stu-id="66db8-141">If the location that must be replenished isn't a fixed location, it's not possible to determine which item to replenish.</span></span> <span data-ttu-id="66db8-142">ولذلك، يجب أن تكون كمية فعلية‬ معينة على الأقل متوفرة قبل حدوث التزويد.</span><span class="sxs-lookup"><span data-stu-id="66db8-142">Therefore, at least some on-hand quantity is required before replenishment occurs.</span></span>
